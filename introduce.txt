一、公共通用页面（所有用户可见）
这类页面无需登录即可访问，主要承担信息展示和基础引导功能。
平台首页
核心功能：展示平台简介、热门待领养宠物轮播、最新救助需求、公益活动公告、合作机构推荐、知识课堂精选；
对应表：pet_info（待领养宠物）、rescue_info（救助需求）、notice_info（公告）、cooperate_org（合作机构）、knowledge_course（知识课程）。
宠物列表页（公开）
核心功能：按宠物类型（猫 / 狗 / 其他）、品种、年龄、健康状态筛选待领养宠物，支持关键字搜索，展示宠物缩略图和基础信息；
对应表：pet_info、pet_image。
宠物详情页（公开）
核心功能：展示宠物完整信息（外观、健康、救助背景）、图片轮播、电子档案摘要、送养条件，提供 “申请领养” 入口；
对应表：pet_info、pet_image、pet_archive、adopt_release。
救助需求列表页（公开）
核心功能：按紧急程度、需求类型（医疗 / 物资 / 暂养）筛选救助项目，展示项目标题、已筹金额、剩余时间，支持关键字搜索；
对应表：rescue_info。
救助需求详情页（公开）
核心功能：展示救助项目详情、宠物情况、需求说明、捐赠进度，提供 “资金捐赠”“物资捐赠”“志愿者报名” 入口；
对应表：rescue_info、pet_info、donate_record、volunteer_service。
知识课堂列表页（公开）
核心功能：按课程类型（救助实操 / 科学饲养）、难度等级筛选课程，展示课程封面、标题、讲师、查看量，支持关键字搜索；
对应表：knowledge_course。
知识课程详情页（公开）
核心功能：展示课程完整内容（图文 / 视频）、讲师简介，提供 “收藏课程”“评论互动” 功能，记录课程查看次数；
对应表：knowledge_course。
公告详情页（公开）
核心功能：展示公告完整内容，支持图文格式，显示发布时间和发布人；
对应表：notice_info。
登录页
核心功能：提供用户名 / 密码登录、验证码登录，支持 “忘记密码”“注册账号” 跳转；
对应表：sys_user。
注册页
核心功能：填写用户基础信息（用户名、密码、手机号、邮箱），完成账号注册，支持用户类型初步选择（普通用户 / 救助者 / 志愿者）；
对应表：sys_user。
忘记密码页
核心功能：通过手机号 / 邮箱验证，重置账号密码；
对应表：sys_user。
关于我们页
核心功能：展示平台简介、公益理念、团队信息、联系方式、合作机构展示；
对应表：cooperate_org。
二、普通用户 / 领养者 / 救助者 / 志愿者（登录后页面）
这类页面需要用户登录后访问，对应不同角色的操作权限，核心是完成个人业务办理。
（一）个人中心模块
个人信息页
核心功能：查看 / 修改个人基础信息（真实姓名、手机号、地址等），上传身份证照片（领养 / 志愿者认证用），修改登录密码；
对应表：sys_user。
我的领养页
核心功能：查看我的领养申请记录（待审核 / 通过 / 驳回）、已领养宠物记录，跟踪领养进度；
对应表：adopt_apply、adopt_record、pet_info。
我的救助页
核心功能：查看我发布的救助需求（待援助 / 援助中 / 已完成）、救助进度、捐赠明细，编辑 / 下架未完成的救助需求；
对应表：rescue_info、donate_record、volunteer_service。
我的送养页
核心功能：发布送养信息（关联已救助宠物）、查看我发布的送养信息（待审核 / 已上线 / 已下架）、审核他人领养申请；
对应表：adopt_release、pet_info、adopt_apply。
我的捐赠页
核心功能：查看我的资金 / 物资捐赠记录，跟踪物资配送状态，查看捐赠凭证；
对应表：donate_record。
我的志愿服务页
核心功能：查看我报名的志愿服务（待执行 / 执行中 / 已完成）、服务时长统计、积分明细，提交服务完成确认；
对应表：volunteer_service、sys_user（爱心积分）。
我的收藏页
核心功能：查看我收藏的宠物、救助项目、知识课程，支持取消收藏；
对应表：pet_info、rescue_info、knowledge_course（收藏关系可通过中间表维护，或前端缓存，建议新增user_collect中间表）。
我的消息页
核心功能：接收平台通知（审核结果、回访提醒、公告推送）、查看消息详情、标记已读；
对应表：可新增sys_message消息表，关联sys_user。
（二）业务操作页面
领养申请提交页
核心功能：填写领养申请信息（自身情况、养宠规划），上传居住证明图片，提交申请；
对应表：adopt_apply、sys_user、adopt_release。
救助需求发布页
核心功能：填写救助需求信息（标题、需求类型、紧急程度、目标金额），关联流浪宠物，上传证明材料，提交审核；
对应表：rescue_info、pet_info。
送养信息发布页
核心功能：选择已救助宠物，填写领养条件、押金金额、回访要求，提交审核；
对应表：adopt_release、pet_info。
捐赠提交页
核心功能：选择捐赠类型（资金 / 物资），填写捐赠金额 / 物资信息，提交捐赠，生成捐赠记录；
对应表：donate_record、rescue_info、pet_info。
志愿者服务报名页
核心功能：选择救助项目，填写服务类型、预计服务时间，提交报名；
对应表：volunteer_service、rescue_info。
投诉举报提交页
核心功能：选择举报对象类型、填写举报内容，上传证据材料，选择是否匿名，提交投诉；
对应表：complaint_report。
三、合作机构用户（登录后页面）
这类页面针对合作机构（宠物医院 / 商家 / 公益组织），主要完成机构业务对接和信息管理。
机构信息页
核心功能：查看 / 修改机构基础信息（地址、联系方式、业务范围），更新资质许可证，提交信息变更审核；
对应表：cooperate_org。
机构托管宠物页
核心功能：查看托管的宠物列表，更新宠物健康状态、医疗记录，提交宠物医疗费用明细；
对应表：pet_info、pet_archive、cooperate_org。
机构捐赠接收页
核心功能：查看针对本机构的物资捐赠记录，确认接收捐赠，填写物流签收信息；
对应表：donate_record、cooperate_org。
机构合作统计页
核心功能：统计机构参与救助的宠物数量、医疗服务次数、物资接收明细，生成统计报表；
对应表：pet_info、rescue_info、donate_record。
四、管理员后台页面（权限最高，需专属登录入口）
这类页面针对平台管理员，主要完成系统配置、内容审核、数据管理、权限管控。
（一）系统管理模块
管理员首页（数据看板）
核心功能：展示平台核心数据（用户总数、宠物总数、救助完成数、领养成功数、捐赠总额），可视化图表统计（用户增长、业务趋势）；
对应表：所有业务表的聚合统计。
用户管理页
核心功能：查询 / 筛选 / 导出用户列表，新增 / 编辑 / 冻结 / 启用用户账号，修改用户类型（如升级为管理员），查看用户详细信息；
对应表：sys_user。
机构管理页
核心功能：审核机构入驻申请，查询 / 筛选合作机构列表，暂停 / 终止机构合作，查看机构资质信息；
对应表：cooperate_org。
系统配置页
核心功能：新增 / 编辑 / 禁用系统配置项（平台名称、默认押金、积分比例），查看配置修改日志；
对应表：sys_config。
权限管理页（可选，进阶功能）
核心功能：创建角色、分配角色权限（如菜单权限、操作权限），关联用户与角色；
对应表：可新增sys_role（角色表）、sys_menu（菜单表）、sys_user_role（用户角色关联表）。
（二）内容审核模块
宠物信息审核页
核心功能：审核用户发布的宠物信息，通过 / 驳回申请，填写审核备注；
对应表：pet_info。
救助需求审核页
核心功能：审核用户发布的救助需求，验证需求真实性，通过 / 驳回申请，填写审核备注；
对应表：rescue_info。
送养信息审核页
核心功能：审核用户发布的送养信息，确认宠物符合送养条件，通过 / 驳回申请，填写审核备注；
对应表：adopt_release。
知识课程审核页
核心功能：审核专家 / 管理员发布的知识课程，确认内容合规性，通过 / 驳回申请，填写审核备注；
对应表：knowledge_course。
公告发布页
核心功能：新增 / 编辑 / 下架公告，设置公告类型、目标用户、是否弹窗展示，预览公告内容；
对应表：notice_info。
（三）业务管理模块
宠物档案管理页
核心功能：查询 / 筛选宠物档案，编辑宠物医疗记录、疫苗记录，导出宠物档案明细；
对应表：pet_archive、pet_info。
领养记录管理页
核心功能：查看所有领养记录，跟踪领养押金状态，提醒管理员进行回访，更新领养回访状态；
对应表：adopt_record、visit_record。
回访记录管理页
核心功能：新增 / 编辑回访记录，分配回访任务给志愿者，查看回访统计报表，处理回访异常情况；
对应表：visit_record、adopt_record。
捐赠管理页
核心功能：查看所有捐赠记录，统计捐赠总额，导出捐赠明细用于公示，核实大额捐赠信息；
对应表：donate_record。
志愿服务管理页
核心功能：查看志愿者服务记录，审核服务完成情况，发放服务积分，统计志愿者服务时长排名；
对应表：volunteer_service、sys_user。
投诉举报处理页
核心功能：查看待处理投诉，受理 / 办结投诉，填写处理结果，跟踪投诉处理进度，统计投诉类型分布；
对应表：complaint_report。
（四）数据统计与报表模块
用户统计页
核心功能：按用户类型、注册时间统计用户数量，生成用户增长趋势图，导出用户统计报表；
对应表：sys_user。
宠物业务统计页
核心功能：按宠物状态、类型统计宠物数量，统计救助完成率、领养成功率，生成业务趋势图；
对应表：pet_info、rescue_info、adopt_record。
公益数据统计页
核心功能：统计捐赠总额、志愿者服务总时长、爱心积分发放总量，生成公益数据可视化报表；
对应表：donate_record、volunteer_service、sys_user。
报表导出页
核心功能：导出各类业务报表（用户报表、宠物报表、捐赠报表），支持 Excel 格式下载，设置报表导出权限；
对应表：所有业务表。
五、页面设计核心原则
角色权限隔离：不同角色看到不同页面（如普通用户无后台管理页面，管理员有全部权限），避免越权操作；
业务流程闭环：每个业务（救助 / 领养）从发起到完成，对应页面流程连贯（如发布救助→对接援助→完成救助）；
数据联动展示：页面数据需关联对应数据库表，确保信息实时同步（如领养申请审核后，用户个人中心同步更新状态）；
操作便捷性：核心业务操作（如提交领养申请、发布救助需求）页面简洁，减少填写项，支持图片上传等便捷功能；
信息可视化：管理员后台多采用图表展示统计数据，方便快速掌握平台运营情况。
